# Security Policy

## Supported Versions

We actively support and provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:               |

## Reporting a Vulnerability

If you discover a security vulnerability, please send an email to moltycollab@gmail.com.

All security vulnerabilities will be promptly addressed.

## What to Include

Please include:
1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

## Response Timeline

- **Initial Response:** 24-48 hours
- **Severity Assessment:** 3-5 days
- **Fix Development:** Based on severity
- **Public Disclosure:** After fix is released

## Security Best Practices

### For Users
- Don't share your API keys
- Use strong passwords
- Report suspicious activity immediately

### For Developers
- Never commit secrets to version control
- Use environment variables for sensitive data
- Follow OWASP guidelines
- Keep dependencies updated

### For Deployments
- Use HTTPS in production
- Implement rate limiting
- Enable CORS restrictions
- Monitor logs for suspicious activity
- Regular security audits

## Sensitive Data

We never collect:
- Passwords (use hashing)
- Credit card information
- Personal identifiable information (PII) beyond what's necessary

## Compliance

We aim to comply with:
- GDPR (data protection)
- OWASP Top 10
- Security best practices

## Updates

This policy may be updated. Check back regularly.
